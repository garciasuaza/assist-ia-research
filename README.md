# Assist-IA-Research redesign

Prototipo moderno para reemplazar o mejorar el bloque embebido actual de Google Sites.

## Archivos

- `index.html`: página autocontenida con HTML, CSS y JavaScript.
- `paper_lifecycle_v3.svg`, `command_table_v2.svg`, `tools_connections_v4.svg`: diagramas del sistema usados en la sección de arquitectura.
- `AI_HEI_Colombia_WorkingPaper.pdf`: copia local del working paper.
- `paper-page-1.png`: preview de la primera página del paper.
- `preview-desktop.png`, `preview-tablet.png`, `preview-mobile.png`: capturas de verificación responsive.

## Cómo usarlo en Google Sites

1. Abre `index.html`.
2. Copia todo el contenido del archivo.
3. En Google Sites, edita la página.
4. Inserta un bloque `Embed` / `Insertar` -> `Embed code`.
5. Pega el código completo.
6. Sube también los SVG y `paper-page-1.png` si quieres mantener las imágenes dentro de Google Sites, o reemplaza sus rutas por URLs públicas de Drive.
7. Ajusta la altura del embed para que no corte el contenido.

## Cambios principales

- Hero con escena visual animada tipo mapa de investigación.
- Nueva sección inicial de arquitectura del sistema con acciones, comandos y diagramas.
- Navegación fija por secciones.
- Métricas animadas.
- Flujo de cinco pasos con pestañas interactivas y prompts fieles del proceso.
- Se eliminó el recuadro de lectura crítica.
- Sección de validación más clara.
- Presentación del paper como artefacto académico con preview visible de la primera página.
- Diseño responsive probado en desktop, tablet y móvil.
